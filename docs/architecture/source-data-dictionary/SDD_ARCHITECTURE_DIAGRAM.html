Source Data Dictionary (SDD) Agent

One specialist capability inside a single custom agent harness. It reconstructs the meaning, structure, keys, code values, evidence and gaps for every in-scope source table and column — for one engagement, one LOB/domain, one allow-listed set of tables.

The core idea: deterministic code establishes facts, the LLM proposes meaning (and must cite evidence), humans approve. Read it top to bottom. Click a legend chip to highlight just those steps.

Deterministic code — facts & rules LLM — semantic judgment Human — review & approval Fail-closed / approval gate

Inputs — the bounded "context envelope"

Everything the agent is allowed to see is assembled before the run. It never scans directories or pulls whole repositories.

The pipeline — 10 phases

0
Scope & context assemblyCodeFail-closed
Lock the run — which engagement, LOB/domain and exactly which tables are allowed. Resolve the one approved knowledge-pack version, pull prior decisions, and build the context envelope. Fails closed if anything essential is missing.
writes: context_snapshotreads: episodic + semantic memory
1
Evidence ingestionCode
Read raw facts only — metadata and approved profiles for the allow-listed tables. Nothing has touched the LLM yet; these are facts, not opinions. Normalized into evidence records.
writes: source_object / attribute_observationprofile_evidence
2
Relationship-candidate detectionCode
Find likely keys and joins. Declared constraints are authoritative; undeclared candidates come from name/type/overlap heuristics, each scored — weak ones are suppressed from the LLM but kept auditable (never silently dropped).
writes: relationship_candidatedeclared | heuristic
3
Object-level semantic analysisLLM
For each table, propose what it means — business name, purpose, subject area, synonyms — each tagged OBSERVED / INFERRED / DECIDED / UNRESOLVED. Anything inferred must cite evidence; insufficient evidence becomes an open question, never a guess.
uses skill: SA1reads: scoped glossary
4
Attribute-level semantic analysisLLM
Same discipline per column: meaning and lifecycle. Coded columns get value meanings from profiles cross-checked to governed code sets (unmapped values flagged, never invented). Privacy-looking columns get a candidate sensitivity flag — always routed to a steward, never finalized here.
skills: SA2, SA3privacy → steward
5
Contradiction & gap detectionCode+ LLM critic
Deterministic checks for term conflicts, superseded decisions and coverage gaps. Then a separate critic challenges plausibility and cross-table consistency. The critic should run on a different model with reduced context — agreement from a shared context counts as correlated, not independent, confirmation.
skill: CR1unresolved → open_question
6
Confidence scoringCode
Report confidence as its components — evidence type, evidence count, critic agreement — not one opaque number. Until calibrated on unseen labelled data it is a triage signal, not a quality guarantee.
no answer-key tuning
7
Contract validationCodeFail-closed gate
Mechanically enforce the quality gates: 100% coverage, nothing INFERRED marked OBSERVED, no evidence-less claim, every key / privacy class / material relationship flagged for review. Failures loop back to phases 3–6 with a bounded repair budget — they never persist as reviewable.
validates against JSON Schema contracts
8
Human review checkpointHumanApproval gate
Targeted review, not exhaustive: keys, privacy, material relationships, every inference, all unresolved items and contradictions are queued; bulk catalog facts are spot-checkable, not queued one-by-one. Humans approve / modify / reject / defer. This is the only path to APPROVED — nothing auto-approves.
skill: X1 prepare-for-reviewwrites: review_decision
9
Persistence & downstream generationCode
Persist versioned Delta records with full provenance (run, context, model/prompt/skill versions). The Excel sheets and the App view are regenerated from these records — never edited independently and re-imported.
feeds: Silver / Gold / STTM

What powers each phase

Memory — three stores

Every durable thing lives in exactly one store. Domain content never enters a skill; engagement data never enters the knowledge layer.

Semantic
Governed knowledge packs
Glossary, code sets, standards. Cross-engagement, versioned, read-only via a fail-closed selector (approved versions only).
Episodic
Engagement decisions & runs
review_decision, open_question, context_snapshot. Engagement-isolated, append-only, never auto-promoted.
Procedural
Skills (SKILL.md)
How-to playbooks — SA1/SA2/SA3, critic CR1, prepare-for-review X1. Reusable, evaluated on unseen cases.

Scope gates & tool permissions are deterministic execution — not memory.

Tools & guardrails

Read facts, write only to solution-owned stores. Permissions live in the harness middleware, not in prompts.

  • Read-only metadata tool — Unity Catalog, scoped to the allow-listed tables.
  • Controlled profiling — load-bearing: powers key detection & code-value inference.
  • select_approved_pack — fail-closed; only APPROVED + runtime-eligible packs admitted.
  • Evidence adapter — extracts from old docs with citations; content treated as data, never instructions.

Hard limits the agent cannot cross:

  • Can't mark a value OBSERVED without Phase-1 evidence behind it.
  • Can't leave a table/column uncovered — UNRESOLVED is allowed, silence is not.
  • Can't finalize a privacy class, an approval, or a knowledge-pack change.
  • Time / cost / token / tool-call / recursion limits enforced by the harness.

Outputs

Authoritative — versioned Delta records
Evidence & governance families: observations, profiles, relationship candidates, open questions, validation findings, review decisions. The single source of truth.
▶
Consumption — Excel + Databricks App
Regenerated from the Delta records for review and portability. Never a second source of truth.

The trust vocabulary — every claim is tagged

OBSERVED
Directly present in source evidence (a fact).
INFERRED
A conclusion the LLM proposes — must cite evidence.
DECIDED
Established by an authorized human decision.
UNRESOLVED
Evidence insufficient/contradictory — raises an open question.
Honest status: contracts, schema-wide metadata capture, DQX profiling, evidence assembly and an approved runtime pack now exist. The positive semantic path still requires engagement authorization, governed context assembly, the LLM harness, critique, human review and publication.
