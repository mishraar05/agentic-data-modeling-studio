# Target reference models

No authorized target reference model is supplied in `v0.6.0`. An admissible reference model must identify product, module, version, scope, owner, effective dates, authorization, and licensing basis before it is registered.

Absence of a target reference model is an explicit knowledge gap. The Agentic Data Analyst and Modeler may create a target model from engagement evidence and requirements as a project deliverable, but it must not present an invented reusable reference model as governed knowledge.
