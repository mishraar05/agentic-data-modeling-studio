# Knowledge completeness assessments

Completeness assessments are versioned governance records linked to an immutable knowledge-pack version. They are not knowledge content and do not change the pack they assess.

## Required interpretation

- Completeness is always measured against an explicit LOB/domain and intended use.
- The overall percentage is a weighted coverage indicator, not a correctness or approval score.
- Runtime eligibility, licensing, correctness, and expert approval remain separate gates.
- Each score must cite present assets and list observable gaps.
- Reassessment creates a new assessment version; prior assessments remain auditable.
